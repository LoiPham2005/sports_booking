# 📘 flutter_base2 — Master Plan

> **Phiên bản kế thừa & cải tiến** của `flutter_base`, được thiết kế lại để **lean — modern — scalable — production-ready**.
> Dựa trên toàn bộ kinh nghiệm đã đúc kết qua `flutter_base` (342 file Dart, 3 năm tích lũy patterns).

---

## 🎯 1. Mục tiêu của flutter_base2

| Mục tiêu | flutter_base (v1) | flutter_base2 (v2) |
|---|---|---|
| **Architecture** | Feature-first + Clean (3 flavors) | Giữ nguyên — đã tối ưu |
| **State Management** | Hybrid: Riverpod + BLoC + Cubit + GetX (legacy) | **Riverpod CHÍNH THỨC** (loại bỏ BLoC/Cubit/GetX khỏi feature) |
| **Flutter SDK** | 3.44.0 / Dart 3.12.0 | **3.44.0 / Dart 3.12.0** (giữ nguyên để ổn định, riverpod_lint tạm disable) |
| **Example features** | 9 sub-features (gây nhiễu) | **1–2 reference pattern** (voucher full) |
| **CI/CD** | Không có | **GitHub Actions + Fastlane** |
| **Testing** | 1 file widget_test stub | **Scaffold đầy đủ** (notifier / service / widget / golden) |
| **Env loading** | Manual qua FlavorConfig | **Envied (typed env)** + FlavorConfig hybrid |
| **Documentation** | CLAUDE.md tổng quát | **CLAUDE.md + ARCHITECTURE.md + CONTRIBUTING.md + skills/** |
| **Code-gen** | 7 builders | Giữ nguyên + tối ưu config |

## 🧠 4. State Management Pattern (BaseNotifier)

Giữ nguyên design của v1 — đã rất tốt:

```dart
@riverpod
class VoucherNotifier extends _$VoucherNotifier with BaseNotifier<List<VoucherModel>> {
  late VoucherService _service;

  @override
  Future<List<VoucherModel>> build() async {
    _service = VoucherService(ref.read(dioProvider));
    return _service.getList();
  }

  Future<void> refresh() => runAsync(
    action: _service.getList,
    keepPreviousOnLoading: true,
  );

  Future<void> search(String q) => runAsync(
    action: () => _service.search(q),
    cancelPrevious: true,
    keepPreviousOnLoading: true,
  );
}
```

**4 methods cốt lõi của BaseNotifier:**

| Method | Khi dùng |
|---|---|
| `runAsync` | Service trả `Future<T>` raw |
| `runUnwrap` | Service trả `Future<ApiResponse<R>>` |
| `runPagination` | Service trả `Future<ApiResponse<ApiPaginatedData<R>>>` |
| `runResult` | Repository trả `Future<Result<R>>` |

**Page pattern (HookConsumerWidget):**

```dart
class VoucherPage extends HookConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final state = ref.watch(voucherProvider);
    final notifier = ref.read(voucherProvider.notifier);
    useAsyncValueListener(provider: voucherProvider, ref: ref);

    return Scaffold(
      body: switch (state) {
        AsyncValue(:final value?, isLoading: true) =>
          Stack(children: [_buildContent(value), const LinearProgressIndicator()]),
        AsyncData(value: final list) when list.isEmpty => const EmptyWidget(),
        AsyncData(:final value) => _buildContent(value),
        AsyncError(:final error) => ErrorRetry(onRetry: notifier.refresh),
        _ => const LoadingWidget(),
      },
    );
  }
}
```

## 🌍 5. Multi-flavor & Environment

3 flavors: **dev / stg / prod**

| Component | dev | stg | prod |
|---|---|---|---|
| `app.name` | MyApp Dev | MyApp Stg | MyApp |
| `applicationId` | com.example.app.dev | com.example.app.stg | com.example.app |
| `apiBaseUrl` | api-dev.example.com | api-stg.example.com | api.example.com |
| `enableLogging` | true | true | false |
| `enableCrashlytics` | false | true | true |

**Env loading hybrid:**

- **Compile-time secrets** (API keys, OAuth secrets): qua `envied` từ `.env.{flavor}` (in .gitignore)
- **Runtime config** (flags, URLs): qua `FlavorConfig` singleton + Firebase Remote Config

| 5 | 1 widget_test stub | **Scaffold test đầy đủ + golden** | Văn hóa test sớm |
| 6 | Env qua FlavorConfig string | **Envied (typed)** | Compile-time safety cho secrets |
| 7 | Makefile có PowerShell | **Bash-only** | Mac/Linux dev-friendly |
| 8 | 5 mason bricks (gây nhiễu) | **2 bricks** (full / lean) | Đỡ phải nhớ chọn cái nào |
| 9 | easy_localization | **flutter_localizations + ARB** | Chuẩn Flutter team, hot reload tốt hơn |
| 10 | 7 ngôn ngữ stub | **2 ngôn ngữ (vi, en)** | Thực dụng — mở rộng khi cần |
